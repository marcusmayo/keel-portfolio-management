# Inbox

Show what is waiting in the triage inbox (documents staged for work-item creation but not yet triaged).

List the contents of knowledge/inbox/. For each file, read its frontmatter (source, ingested) and
give a one-line summary of what the document covers. These are PENDING triage — they have not yet
produced work items.

If knowledge/inbox/ is empty, say the triage inbox is empty.

End with a count and a reminder that /process triages everything pending here. This is read-only;
never create work items from this command — that is what /process does.
